## 2. 产品结果

完成后，AI 页面以“回答”为视觉中心：

1. 删除左侧整高说明栏，释放桌面端约 210–230px 横向空间。
2. 在对话区上方使用一条紧凑信息行，保留“全站考试事实查询”和必要的使用提示。
3. 将模型、隐私和数据边界说明移到页面底部的折叠详情中，默认不占用回答空间。
4. 输入框保持稳定的普通尺寸，不提供单独放大按钮，也不允许拖拽输入框或整个对话外框。
5. “全屏查看回答”进入回答优先模式：回答区获得主要高度，标题栏和输入区保持紧凑，支持 `Escape` 退出并恢复触发按钮焦点。
6. 每条已完成的助手回答都可导出 PNG、复制富文本、下载独立 HTML；未完成或错误回答不提供误导性的正式导出。
7. 网络失败、限流、流中断、空回答、用户停止和导出失败都有可区分、可恢复、可读屏的状态。

## 3. 组件与代码结构

### 3.1 页面布局

修改 `src/pages/AIAssistantPage.tsx`：

- 将当前两列 `side rail + chat` 改为单列布局。
- 新增窄信息行，内容限制为页面名称、用途短句和当前检索范围提示；在 390px 下自然换行，不产生横向滚动。
- 页面主体宽度沿用现有 1440px 上限，但对话区占满可用宽度。
- 在对话组件之后放置默认折叠的 `<details>`，承载数据范围、DeepSeek/AQA 路由、会话存储与不发送资料说明。
- 禁用状态仍显示现有“内部验收”面板，不改变功能开关。

### 3.2 回答工作区

重构 `src/components/ai/AIChatPanel.tsx`：

- 删除 `inputExpanded` 状态、输入框放大/缩小按钮及相应样式。
- 删除外层 `resize-y`；普通模式使用稳定的响应式高度，回答区自身滚动。
- 输入框保持固定最小高度并允许内部滚动；保留 `Enter` 发送、`Shift + Enter` 换行和 2,000 字限制。
- 将 `panelExpanded` 重命名为表达明确的回答专注状态，例如 `answerFocusMode`。
- 专注模式使用带 `role="dialog"`、`aria-modal="true"` 和明确标题的全屏层；回答区 `flex: 1`，页头、建议和输入区压缩为辅助区域。
- 进入专注模式时记录触发按钮，退出后恢复焦点；支持 `Escape`，并继续锁定背景滚动。
- 长回答期间只在用户已接近底部时自动跟随；用户向上阅读时不强制跳回底部。

建议拆分以下小组件，避免继续扩大 `AIChatPanel`：

- `AIAnswerCard`：回答、引用、状态和单条导出操作。
- `AIAnswerActions`：PNG、富文本、HTML 三种导出入口与反馈。
- `AIAssistantDisclosure`：底部折叠说明。
- `AIResponseState`：失败、空回答、停止和重试状态。

### 3.3 导出模块

新增 `src/components/ai/exportAIAnswer.ts`，导出对象只接受已完成的助手消息、当前检索范围和已经过允许列表校验的引用。

统一导出视图模型：

```ts
interface AIAnswerExportModel {
  messageId: string;
  markdown: string;
  plainText: string;
  renderedHtml: string;
  citations: AICitation[];
  contextLabels: string[];
  exportedAt: string;
  completionState: "complete" | "incomplete";
}
```

安全约束：

- 不直接信任或执行模型 HTML；继续以 `react-markdown` 的 `skipHtml` 结果为基础。
- HTML 文件不包含脚本、事件属性、远程字体或外部追踪资源。
- 链接只保留 `http/https`，并添加安全的 `rel` 属性。
- 导出只包含当前单条回答、来源、检索范围、导出时间和 ExamBridge 品牌，不包含整段会话中的其他用户问题。

三种导出：

1. **PNG**
   - 复用已安装的 `html2canvas`，对隔离的导出卡片进行渲染，不直接截图整个页面。
   - 普通回答下载单个 `exambridge-answer-YYYYMMDD-HHmm.png`。
   - 超出安全画布高度时按语义块分页，下载带 `-01`、`-02` 编号的多张 PNG，并先向用户说明页数。
   - PNG 固定浅色背景、足够留白、清晰来源区和 2x 像素密度；不受当前暗色/滚动位置影响。
2. **复制富文本**
   - 优先通过 `ClipboardItem` 同时写入 `text/html` 与 `text/plain`。
   - 浏览器不支持富文本剪贴板时降级为纯文本，并明确提示降级结果。
3. **下载 HTML**
   - 生成 UTF-8、可离线打开、带内联 CSS 的独立 HTML 文件。
   - 文件包含标题、回答正文、来源和核验状态，不包含应用脚本或会话状态。

### 3.4 状态与恢复

将当前单一 `pending/error` 表示扩展为明确状态：

```ts
type AIAnswerState =
  | "streaming"
  | "complete"
  | "stopped"
  | "interrupted"
  | "rate-limited"
  | "service-unavailable"
  | "empty";
```

行为规则：

- `stopped`：保留用户主动停止前的片段，标注“未完成”，不显示正式导出操作；可重新发送上一问题。
- `interrupted`：保留已收到内容并显示“连接中断”，提供重试；不得把片段伪装成完整答案。
- `rate-limited`：显示服务器返回的等待秒数，并在倒计时结束前禁用该重试按钮。
- `service-unavailable`：保留问题与已接收片段，提供重试和复制问题。
- `empty`：当收到结束事件但答案为空时显示专门空状态，不渲染空白气泡。
- 导出错误不会改变回答本身，只在回答卡操作区显示可关闭反馈。
- 重试复用原始用户问题和当时的 scopes，不把错误文案加入模型对话历史。

验收：任何失败都不会留下永久 loading、空白回答或可被误认为完整结论的片段。

## 7. 风险与控制

- **PNG 高度和内存**：采用语义分页、像素上限和显式页数提示，避免浏览器崩溃。
- **剪贴板兼容性**：富文本失败自动降级纯文本，不宣称复制了未写入的格式。
- **导出内容注入**：复用安全 Markdown 渲染和链接允许列表，独立 HTML 永不带脚本。
- **全屏焦点陷阱**：实现 Escape 和焦点恢复，并用可访问对话语义测试。
- **流式回答竞态**：只允许 complete 状态导出；中断片段保留但明确标识。
- **PWA 更新回归**：不改变 Service Worker；仍运行离线刷新核心回归。
